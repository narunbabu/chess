# 2026-06-18 — Admin user tooling + security remediation (M1/M6/M7/M8/L3-L8)

## Admin dashboard: user visibility + custom email
Surfaced more user data to admins and added an outbound email tool.

- **Backend** (`AdminDashboardController`):
  - `overview()` user list now selects `mobile_country_code`, `mobile_number`, `last_login_at` (joined/last-active were already returned).
  - `userDetail()` profile now includes `mobile_country_code`, `mobile_number`, `mobile_verified_at`, `last_login_at`.
  - New `sendEmail()` → `POST /admin/dashboard/user/{id}/email` (subject ≤200, body ≤5000). Queues `AdminCustomMail`; org admins are scoped to their own org; logs identifiers only.
  - New `App\Mail\AdminCustomMail` (ShouldQueue, `emails` queue) + `resources/views/emails/admin-custom.blade.php` (body escaped + `nl2br`, no HTML injection).
  - No migration — all columns already existed.
- **Frontend** (`AdminDashboard.js`):
  - Users table gained a sortable **Joined** column (alongside Last Active).
  - User detail panel shows **mobile number** (+ verified badge), **last active**, and **last login**.
  - **✉ Email** button opens an inline composer (subject + body) posting to the new endpoint with success/error feedback.
- **Test**: `tests/Feature/AdminUserEmailTest.php` — admin queues mail, non-admin 403, invalid 422, org-admin cross-org 403.
